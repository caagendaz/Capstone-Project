# Data Directory Structure

This directory contains all datasets for the E. coli antibiotic resistance project.

## Directory Organization

```
data/
├── raw/                # Original, unmodified data from Kaggle
├── processed/          # Cleaned, merged, and ready-to-model datasets
└── external/           # Genomic data from BVBRC API
```

## File Placement Guide

### raw/ Directory

**What goes here**: Original CSV files from Kaggle

**Expected files**:
- `ecoli_resistance.csv` - Main resistance phenotype dataset
  - Columns: isolate_id, antibiotic, mic_value, resistance_phenotype, testing_standard
  - Source: Kaggle dataset (download using `data_acquisition.py`)
  - Size: ~12,780 rows

**How to get**:
```python
from src.data_acquisition import KaggleDataDownloader
downloader = KaggleDataDownloader(output_dir="data/raw")
downloader.download_dataset("username/ecoli-resistance-dataset")
```

Or manually:
1. Download from Kaggle website
2. Place CSV file in `data/raw/`
3. Rename to `ecoli_resistance.csv`

---

### external/ Directory

**What goes here**: Genomic and AMR gene data from BVBRC

**Expected files**:
- `amr_genes.csv` - AMR gene presence/absence data
  - Columns: isolate_id (or genome_id), gene_name, presence
  - Generated by: `BVBRCDataDownloader.get_amr_genes()`
  
- `genome_metadata.csv` - Genome information
  - Columns: genome_id, organism_name, isolation_source, etc.
  - Generated by: `BVBRCDataDownloader.search_genomes()`

- `*.fasta` (optional) - Genome sequences
  - Generated by: `BVBRCDataDownloader.download_fasta()`

**How to get**:
```python
from src.data_acquisition import BVBRCDataDownloader

bvbrc = BVBRCDataDownloader(output_dir="data/external")

# Get genome metadata
genomes_df = bvbrc.search_genomes("Escherichia coli", limit=1000)
genomes_df.to_csv("data/external/genome_metadata.csv", index=False)

# Get AMR gene data
genome_ids = genomes_df['genome_id'].tolist()
amr_df = bvbrc.get_amr_genes(genome_ids)
bvbrc.save_amr_data(amr_df, "amr_genes.csv")
```

### processed/ Directory

**What goes here**: Cleaned and merged datasets ready for modeling

**Expected files** (automatically generated by preprocessing):
- `merged_data_CLSI.csv` - Data using CLSI testing standard
  - Phenotype + genotype merged
  - Binary resistance labels added
  - Ready for modeling
  
- `merged_data_EUCAST.csv` - Data using EUCAST testing standard
  - Same structure as CLSI
  - Separate due to different interpretation criteria

**Structure**:
```
Columns:
- isolate_id: Unique identifier
- antibiotic: Antibiotic name (e.g., CIPROFLOXACIN)
- mic_value: Minimum Inhibitory Concentration
- resistance_phenotype: S, I, or R
- testing_standard: CLSI or EUCAST
- binary_resistant: 0 (susceptible) or 1 (resistant)
- binary_resistant_liberal: Alternative binary encoding
- gene_1, gene_2, ..., gene_N: Binary (0/1) for AMR gene presence
```

**How generated**:
```python
from src.data_preprocessing import EcoliDataPreprocessor

preprocessor = EcoliDataPreprocessor()

# Load raw data
resistance_df = preprocessor.load_resistance_data("data/raw/ecoli_resistance.csv")
genomic_df = preprocessor.load_genomic_data("data/external/amr_genes.csv")

# Process
resistance_df = preprocessor.clean_resistance_data(resistance_df)
splits = preprocessor.split_by_testing_standard(resistance_df)

for standard, df in splits.items():
    isolates = df['isolate_id'].unique()
    features = preprocessor.build_amr_feature_matrix(genomic_df, isolates)
    merged = preprocessor.merge_phenotype_genotype(df, features)
    merged = preprocessor.create_binary_labels(merged)
    preprocessor.save_processed_data(merged, f"merged_data_{standard}.csv")
```

## Data Requirements

### Minimum Requirements
- At least 100 samples per antibiotic
- Both phenotype and genotype data for each isolate
- Valid resistance phenotypes (S, I, or R)
- At least some AMR gene annotations

### Recommended
- 1,000+ isolates per antibiotic
- Multiple antibiotics (5-10)
- Coverage of both CLSI and EUCAST standards
- Diverse AMR gene profiles (50+ genes)

## Troubleshooting

### Issue: "File not found"
- Ensure data is downloaded to correct directory
- Check file names match expected names
- Verify paths in config.py

### Issue: "Empty dataset after merging"
- Check that isolate_ids match between datasets
- Verify genomic data covers same isolates as phenotype data
- May need to use genome_id → isolate_id mapping

### Issue: "Too few samples"
- Reduce `MIN_SAMPLES_PER_ANTIBIOTIC` in config.py
- Focus on most common antibiotics only
- Consider combining similar antibiotics

### Issue: "Memory error"
- Process data in chunks
- Filter to specific antibiotics early
- Use sparse matrices for gene presence/absence

## Data Sources

### Kaggle
- **URL**: https://www.kaggle.com/datasets/
- **Search**: "E. coli antibiotic resistance"
- **Requirements**: Kaggle account, API credentials
- **Format**: CSV with MIC values and phenotypes

- **API**: https://www.bv-brc.org/api/

- **Format**: JSON (converted to CSV by scripts)
